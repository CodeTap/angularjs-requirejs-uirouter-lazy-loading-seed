This is an example single page application written in AngularJS and Ui-Router. It uses RequireJS and implements lazy loading. The About Page contains child views and multiple named views, and the Contact page contains only named views.

Testing the App

The App comes complete with karma test runners and protractor. Karma will run several unit test where protractor the protractor tests will make sure the transition between states happens properly. It also uses Karma's coverage plug-in so you can see the amount of code covered by unit tests.

Karma unit tests are run using the following command.

npm run test

Protractor End to End tests are run using the following command.

npm run protractor

Why UI Router?

Ui-router is more powerful for larger scale applications than normal ngRoute. The ability to do the following separates it as the easy choice:

  • ui-router allows for nested views and multiple named views. This is very useful with larger app where you may have pages that inherit from other sections.
  • states allow you to map and access different information about different states and you can easily pass information between states via $stateParams.
  • ui-router allows for you to have strong-type linking between states based on state names. Change the url in one place will update every link to that state when you build your links with ui-sref. Very useful for larger projects where URLs might change.

Combining ui-router with lazy loading just makes creating a large scale angular application easier to manage. Instead of having to declare all dependencies within the head of every file requirejs can do that heavy lifting for you.
